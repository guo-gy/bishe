# 基于跨模态对齐的广义指代图像分割算法研究及应用

这是一份项目内部总文档，后续围绕本毕设的方案、文献、实验、实现和论文写作都尽量在这一份文档内持续补充，不再拆分成多个独立计划文件。

## 1. 当前项目定位

本项目以 `CRIS: CLIP-Driven Referring Image Segmentation (CVPR 2022)` 作为核心 baseline，围绕任务书中最适合落地、也最容易形成论文创新表述的两个方向展开：

1. 多模态平衡
2. 自蒸馏

目标不是机械复现或直接照搬已有论文，而是在公开有效方法的基础上，设计一个结构上能讲清楚、实验上能跑出结果、论文里能合理表述为“改进方法”的方案。

整体主线暂定为：

`CRIS baseline -> 加入多模态平衡模块 -> 加入跨模态自蒸馏训练策略 -> 在标准 RIS 数据集和尽可能贴近广义场景的数据设置上验证 -> 完成论文与系统实现`

## 2. 为什么选 CRIS 作为 baseline

选择 `CRIS` 的原因：

1. 它是 CLIP 驱动的经典 RIS 方法，作为对比基线有说服力。
2. 结构相对清晰，便于后续插入改进模块。
3. 视觉编码器和文本编码器已经成熟，适合作为跨模态改进实验平台。
4. 如果后面要写论文，“基于 CRIS 的改进方法”会比完全重起炉灶更稳。

当前默认策略：

1. 先复现 `CRIS`
2. 再做单模块改进
3. 最后做组合模型和消融实验

## 3. 本课题拟采用的核心创新点

### 3.1 创新点一：多模态平衡

这个方向主要解决传统 RIS 中“语言指导视觉较多，但视觉反向约束语言较弱”的问题。很多方法本质上是文本单向引导视觉，导致在复杂语义、细粒度属性、多实例相似目标场景下，视觉和语言信息融合不够均衡。

本项目中，多模态平衡的目标是：

1. 缓解视觉模态与语言模态交互不对称问题
2. 提升模型对长表达、复杂属性和细粒度目标的理解能力
3. 为多目标表达或空目标表达的后续扩展打基础

拟借鉴的有效思路：

1. 双向跨模态交互，而不是只做文本到视觉
2. 多尺度语义对齐
3. 视觉到文本、文本到视觉两个方向的语义约束
4. 对相似实例进行更强的区分建模

重点参考：

1. `Beyond One-to-One: Rethinking the Referring Image Segmentation (ICCV 2023)`
2. `Prompt-Driven Referring Image Segmentation with Instance Contrasting (CVPR 2024)`

在本项目中的落地设想：

1. 保留 `CRIS` 主体编码结构
2. 在跨模态融合位置增加一个轻量级双向交互模块
3. 让视觉特征和文本特征进行更平衡的双向更新
4. 尽量避免引入过重结构，保证后续系统实现可控

论文中可表述为：

“设计双向跨模态平衡模块，在文本到视觉和视觉到文本两个方向上实现语义交互，缓解多模态表示不均衡问题，提高模型对复杂指代表达的语义理解与分割能力。”

### 3.2 创新点二：自蒸馏

这个方向主要解决细粒度对齐不足、边界模糊、过分割和欠分割等问题，同时尽量不增加推理阶段开销。

拟采用的原则：

1. 不额外引入大型 teacher 模型
2. 使用训练期辅助分支做自蒸馏
3. 测试阶段只保留主干，减少部署负担

重点参考：

1. `CM-MaskSD: Cross-Modality Masked Self-Distillation for Referring Image Segmentation`

在本项目中的落地设想：

1. 保留 `CRIS` 主分支作为标准预测路径
2. 增加 masked branch 或 consistency branch
3. 对视觉 token 或文本 token 做掩码扰动
4. 通过一致性损失让辅助分支向主分支靠拢，或让两者互相约束
5. 推理阶段关闭蒸馏分支，只保留主模型

论文中可表述为：

“引入跨模态自蒸馏训练策略，在训练阶段构建辅助一致性约束，增强视觉与语言的细粒度对齐能力，从而改善目标边界质量并降低过分割与欠分割现象。”

## 4. 当前推荐的整体模型方案

当前最稳妥的方案不是做一个非常激进的新网络，而是在 `CRIS` 上做两步式增强。

### 4.1 主干结构

1. 图像编码器：沿用 `CRIS` 的 CLIP visual encoder
2. 文本编码器：沿用 `CRIS` 的 CLIP text encoder
3. 跨模态融合：在现有融合阶段插入多模态平衡模块
4. 分割解码头：基于 `CRIS` 解码部分适配修改
5. 训练辅助分支：加入自蒸馏分支，但不影响推理路径

### 4.2 可暂定的模型名

内部开发阶段可暂定为：

`CRIS-BSD`

其中：

1. `B` 代表 `Balance`
2. `SD` 代表 `Self-Distillation`

这个名字只是内部代号，后续论文正式命名可以再改。

### 4.3 当前最适合的技术定位

从本科毕业论文角度，最合理的定位不是“提出全新范式”，而是：

1. 基于 `CRIS` 的跨模态对齐增强方法
2. 面向广义指代场景的双向平衡与自蒸馏改进模型
3. 兼顾实验性能与系统可部署性的轻量化实现方案

## 5. 广义指代图像分割这一题目如何落地

这里要特别注意，题目写的是“广义指代图像分割”，所以论文不能完全只停留在传统 one-to-one RIS 上。即使主要实验还是从 `RefCOCO`、`RefCOCO+`、`RefCOCOg` 起步，后面也要尽量补充对广义场景的说明。

广义场景至少包括：

1. 一句话对应单个目标
2. 一句话对应多个目标
3. 一句话在图中没有对应目标

因此当前建议是：

1. 把 `CRIS` 复现和改进实验先在标准 RIS 数据集上跑通
2. 在论文方法和实验章节中明确指出传统 RIS 与广义 RIS 的差别
3. 后续再评估是否补充多目标或空目标设置，至少在写作上把这一点论证清楚

如果后续时间允许，应尽量加入更贴近 generalized RIS 的数据设置或测试方案。

## 6. 实验总路线

### 6.1 第一阶段：复现 CRIS baseline

目标：

1. 获取代码
2. 搭建运行环境
3. 准备数据集
4. 跑通训练与测试流程
5. 记录 baseline 指标

这一阶段必须完成，因为后面所有改进都要建立在可运行 baseline 上。

### 6.2 第二阶段：加入多模态平衡模块

目标：

1. 设计一个轻量级双向跨模态平衡模块
2. 插入到 `CRIS` 合适的融合位置
3. 保证训练能够稳定收敛
4. 与 baseline 做单模块对比

核心实验：

1. `CRIS`
2. `CRIS + Balance`

### 6.3 第三阶段：加入自蒸馏策略

目标：

1. 设计训练期自蒸馏辅助分支
2. 确定蒸馏损失和权重
3. 验证对细节边界和总体指标的提升

核心实验：

1. `CRIS`
2. `CRIS + Self-Distillation`

### 6.4 第四阶段：组合模型与消融实验

目标：

1. 验证两个模块是否互补
2. 分析提升来源
3. 输出论文核心实验表格

核心实验：

1. `CRIS`
2. `CRIS + Balance`
3. `CRIS + Self-Distillation`
4. `CRIS + Balance + Self-Distillation`

### 6.5 第五阶段：系统实现

在算法效果稳定后，再做轻量化演示系统，功能可以控制在：

1. 图像输入
2. 文本输入
3. 分割结果可视化
4. 掩码叠加显示
5. 推理耗时展示

系统不必做得很复杂，但要能支撑毕设答辩展示。

## 10. 当前明确的写作边界

为避免后续论文表述踩坑，当前先定下几个原则：

1. 可以借鉴公开论文中的有效思想，但不能把别人的核心模块直接写成自己原创。
2. 论文表述应强调“基于 baseline 的组合式改进和面向任务需求的适配设计”。
3. 如果最终 generalized 部分实验不够充分，论文中要诚实表述当前工作更偏向“面向广义场景的扩展研究”而不是完全解决所有 generalized setting。
4. 系统实现部分以可演示、可运行、可视化为目标，不追求工程平台级复杂度。
